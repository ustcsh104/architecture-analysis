# 架构分析输出模板

## 产物 1：架构分层图（SVG）

使用 `baoyu-diagram` skill 生成。 根据 Step 3 收集的分层判定结果，描述以下结构：

- 分层标注：🟢接入层 / 🟡业务层 / 🔵持久层 / ⚪工具层 / 🟣扩展层 / 🔘配置层
- 模块间依赖关系（箭头方向=调用方向）
- 横切模块标注（日志/监控/鉴权等贯穿全项目）
- 标注违规跨层调用（红色虚线箭头）

调用方式：对 baoyu-diagram skill 说"画一个架构分层图"，提供项目名和 Step 3 的分层结果。

## 产物 3：典型数据流图（SVG）

使用 `baoyu-diagram` skill 生成至少 3 条核心链路流转图。

### 链路类型

1. **同步请求链路**：HTTP 请求全流程
2. **异步任务链路**：消息消费/定时任务流程
3. **系统启动链路**：初始化全流程

每条链路图包含：
- 每个节点标注数据结构类型（DTO/Entity/Model）
- 横切拦截点标注（鉴权/日志/缓存/限流）
- 异步分支标注

调用方式：对 baoyu-diagram skill 说"画一个数据流图"，提供链路追踪结果。

## 产物输出路径

所有产物统一保存到**工作区根目录**下的 `.qoder/architecture-analysis/` 目录（而非项目实际根目录）：

```
<WORKSPACE_ROOT>/.qoder/architecture-analysis/
├── 01-architecture-layers.svg      # 架构分层图
├── 02-directory-responsibilities.md # 目录职责清单
├── 03-data-flow-sync.svg           # 同步请求数据流图
├── 03-data-flow-async.svg          # 异步任务数据流图
├── 03-data-flow-startup.svg        # 启动流程数据流图
└── 04-architecture-assessment.md   # 架构评估报告
```

**⚠️ 重要说明**：
- **不要**将产物保存到项目实际根目录下（路径可能含中文字符导致编码问题）
- **不要**将产物散落到不同路径（用户难以找到）
- 使用 Write 工具写入 Markdown 文件，使用 baoyu-diagram skill 生成 SVG
- SVG 生成后需确认文件实际保存位置，如果不一致需用 Bash 移动到统一路径

**路径变量约定**：
- `WORKSPACE_ROOT`：Qoder 工作区根目录（产物输出基准路径）
- `PROJECT_ROOT`：项目实际根目录（分析操作基准路径，可能含中文字符）

产物输出路径始终为：`{WORKSPACE_ROOT}/.qoder/architecture-analysis/`
